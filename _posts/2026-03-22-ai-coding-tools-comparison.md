# 2026 我个人的 AI 编码工具链分层选择

我把目前使用的 AI 编码工具按**从轻到重**、**从简单到复杂**分成了四层，来讲讲我的真实使用体验和取舍。

### 第一层：基础 IDE 方案

**VSCode + 各种插件 + GitHub Copilot**

这是大多数人的起点。Copilot 作为辅助写码工具已经非常成熟，搭配 Continue、Codeium 等插件后，日常开发效率不错。

**优点**：生态最成熟、免费方案多、上手简单  
**缺点**：本质还是「单人辅助工具」，无法实现真正的多 agent 协作和工程化流程。

### 第二层：进阶 AI 原生 IDE

**Cursor、Trae、Windsurf**

这一层已经把 AI 深度融入 IDE 本身，体验比传统 VSCode + Copilot 更进一步。Cursor 目前用的人最多，Trae 和 Windsurf 也在快速迭代。

它们适合**单人高强度写码**，但当你需要**同时运行多个 agent**、要做**分支隔离**和**系统化 PR 流程**时，就开始显得力不从心。

### 第三层：命令行代理工具

**Claude Code、Codex、Kimi-CLI 等**

这一层进入了 CLI 时代。我认真用过 Claude Code、Codex、Kimi-CLI。

比起ide，cli更轻更快，而且是厂家直连，不停迭代，开发从来没这么被宠过。单个任务它们表现不错，能写代码、跑命令、改文件。但一旦进入**多 agent 并行**、**工作空间隔离**、**跨 agent 变更比较**、**从 Issue 到 PR 的完整协作流程**时，它们就明显不够用了——缺少工程化的编排能力和统一管理视图。

### 第四层：重型多 agent 协作平台

当单机 IDE 和单 agent CLI 不够用时，我就来到了这一层。

这一层的工具包括：**AMP.sh、Conductor、Superset、cmux**。

**AMP.sh** 的体验是：**花钱最快**。五美金十分钟就烧完了，只能 pay as you go，无法订阅，效果也一般，可能是我给的钱还不够多。最终我把它从工具链里移除了。

目前我重点在 **Conductor、Superset、cmux** 三者之间做选择。

## 三工具核心对比（2026 个人视角）


| 工具                              | 定位              | 支持 OS       | 核心优势                                         | 最适合场景            | 个人评价        |
| ------------------------------- | --------------- | ----------- | -------------------------------------------- | ---------------- | ----------- |
| **Superset**                    | 重度多代理协作平台       | **跨平台**     | CLI 最全、跨代理 diff、浏览器内置、PR 流程                  | 重度多代理、需要强变更管理和审核 | 功能最强，但较重    |
| **cmux**                        | 现代 macOS 终端多工工具 | **仅 macOS** | 极轻、垂直分页、美观 UI、通知系统                           | 日常终端管理、多会话快速监控   | **日常使用最舒服** |
| **Conductor** (conductor.build) | PR 导向代理编排器      | **仅 macOS** | Git worktree 隔离、Dashboard、**issue 到 PR 一条龙** | 中大型项目、重视完整协作流程   | **PR 流程最顺** |


## 我的最终选择建议

**我目前的实际用法**是：

- **cmux**：作为 macOS 下的日常主力终端（最省内存、最舒服）
- **Conductor**：负责重要项目的「Issue → 多 agent 并行 → PR 合并」完整流程
- **Superset**：在需要极致并行能力、强大 diff 功能，或跨 Linux/Windows 环境时使用

**给不同人群的建议**：

- **纯 macOS 重度使用者**：强烈建议三个都试一天（cmux 看舒适度、Conductor 看流程、Superset 看极限能力）
- **需要跨平台**：优先选择 **Superset**
- **专注 PR 工程化流程**：**Conductor** 体验最好
- **追求日常轻量终端体验**：**cmux** 是目前最舒服的选择

---

**总结**

从 VSCode + Copilot 开始，一路试到 Cursor、Claude Code，最后来到 Superset、cmux、Conductor 这个层级，我越来越清楚：**不同阶段需要不同层级的工具**。

单人写码用 Cursor 就够了；要做系统性多 agent 协作和工程化交付，则必须进入第四层。

工具没有绝对的最好，只有最适合自己当前工作流的选择。
